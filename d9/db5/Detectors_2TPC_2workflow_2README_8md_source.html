1 # DPL workflows for the TPC
2 
3 ## TPC reconstruction workflow
4 The TPC reconstruction workflow starts from the TPC digits, the *clusterer* reconstructs clusters. Those are
5 *converted* to RAW pages and passed onto the *decoder* providing the decoded (native) cluster format to the
6 *tracker*.
7 
8 The workflow consists of the following DPL processors:
9 
10 * `tpc-digit-reader` -> using tool [o2::framework::RootTreeReader](../../../Framework/Utils/include/Utils/RootTreeReader.h)
11 * `tpc-clusterer` -> interfaces [o2::TPC::HwClusterer](../reconstruction/include/TPCReconstruction/HwClusterer.h)
12 * `tpc-cluster-converter`-> implements conversions to raw pages, to be moved to a worker class
13 * `tpc-cluster-decoder` -> interfaces [o2::TPC::HardwareClusterDecoder](reconstruction/include/TPCReconstruction/HardwareClusterDecoder.h)
14 * `tpc-tracker` -> interfaces [o2::TPC::TPCCATracking](reconstruction/include/TPCReconstruction/TPCCATracking.h)
15 * `tpc-track-writer` -> implements simple writing to ROOT file
16 
17 MC labels are passed through the workflow along with the data objects and also written together with the
18 output at the configured stages (see output types).
19 
20 ### Input data
21 The input can be created by running the simulation (`o2sim`) and the digitizer workflow (`digitizer-workflow`).
22 The digitizer workflow produces the file `tpcdigits.root` by default, data is stored in separated branches for
23 all sectors.
24 
25 The workflow can currently start from the digits or the clusters, or directly attached to the `digitizer-workflow`,
26 see comment on inputs types.
27 
28 ### Quickstart running the reconstruction workflow
29 The workflow is implemented in the `tpc-reco-workflow` executable.
30 
31 Display all options
32 ```
33 tpc-reco-workflow --help
34 ```
35 
36 Important options for the `tpc-digit-reader` as initial publisher
37 ```
38 --infile arg Name of the input file
39 --treename arg (=o2sim) Name of the input tree
40 --digitbranch arg (=TPCDigit) Digit branch
41 --mcbranch arg (=TPCDigitMCTruth) MC label branch
42 ```
43 
44 The `tpc-cluster-reader` uses the same options except the branch name configuration
45 --clusterbranch arg (=TPCCluster) Cluster branch
46 --clustermcbranch arg (=TPCClusterMCTruth) MC label branch
47 
48 Options for the `tpc-track-writer` process
49 ```
50 --outfile arg (=tpctracks.root) Name of the input file
51 --treename arg (=o2sim) Name of output tree
52 --track-branch-name arg (=TPCTracks) Branch name for TPC tracks
53 --trackmc-branch-name arg (=TPCTracksMCTruth) Branch name for TPC track mc labels
54 ```
55 
56 Examples:
57 ```
58 tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-15
59 ```
60 
61 ```
62 tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-15 --disable-mc 1
63 ```
64 
65 ### Global workflow options:
66 ```
67 --input-type arg (=digits) digitizer, digits, clusters, raw, decoded-clusters
68 --output-type arg (=tracks) digits, clusters, raw, decoded-clusters, tracks
69 --disable-mc arg (=0) disable sending of MC information
70 --tpc-lanes arg (=1) number of parallel lanes up to the tracker
71 --tpc-sectors arg (=0-35) TPC sector range, e.g. 5-7,8,9
72 ```
73 
74 #### Input Type
75 Input type `digitizer` will create the clusterers with dangling input, this is used
76 to connect the reconstruction workflow directly to the digitizer workflow.
77 
78 All other input types will create a publisher process reading data from branches of
79 a ROOT file. File and branch names are configurable. The MC labels are always read
80 from a parallel branch, the sequence of data and MC objects is assumed to be identical.
81 
82 #### Output Type
83 The output type selects up to which final product the workflow is executed. Multiple outputs
84 are supported in order to write data at intermediate steps, e.g.
85 ```
86 --output-type clusters,tracks
87 ```
88 
89 MC label data are stored in corresponding branches per sector. The sequence of MC objects must match
90 the sequence of data objects.
91 
92 #### Parallel processing
93 Parallel processing is controlled by the option `--tpc-lanes n`. The digit reader will fan out to n processing
94 lanes, each with clusterer, converter and decoder. The tracker will fan in from the parallel lanes.
95 
96 #### TPC sector selection
97 By default, all TPC sectors are processed by the workflow, option `--tpc-sectors` reduces this to a subset.

101 #### TPC CA tracker
102 The [tracker spec](src/CATrackerSpec.cxx) interfaces the [o2::TPC::TPCCATracking](reconstruction/include/TPCReconstruction/TPCCATracking.h) worker class which can be initialized using an option string. The processor spec defines the option `--tracker-option`. Currently, the tracker should be run with options
103 ```
104 --tracker-option "refX=83 cont"
105 ```
106 
107 The most important tracker options are:
108 ```
109 cont activation of continuous mode
110 refX= reference x coordinate, tracker tries to propagate all track to the reference
111 bz= magnetic field
112 ```
